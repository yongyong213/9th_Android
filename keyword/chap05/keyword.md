- Thread란 무엇일까요?
  - Thread란 무엇일까요?
    - 프로세스 내에서 순차적으로 실행되는 최소 단위의 실행 흐름
    - 프로세스가 음식점이라면 스레드는 그 음식점의 요리사 느낌
    - 메인 스레드는 UI(사용자 인터페이스)와 관련된 일만 한다
    - 하지만 메인 스레드는 하나 뿐이라 시간이 오래 걸리는 작업을 하게 되면 앱이 멈춘 것과 같은 현상이 생김
  - Thread를 사용하는 예시는 무엇이 있을까요?
    - 화면에 버튼이나 텍스트 그리기
    - 사용자의 터치 이벤트 처리
    - 화면 애니메이션 보여주기
- Main Thread와 Worker Thread
  - View Control을 담당하는 Thread는 무엇일까요?
    - Main Thread
    - 모든 뷰를 제어한다
  - Worker Thread는 어떤 작업을 담당할까요?
    - 네트워크 통신
      - 서버에서 데이터 받아오기
      - 대용량 이미지나 파일 다운로드/업로드
    - 데이터베이스 작업
      - 데이터 조회, 저장, 수정
    - 복잡한 연산 및 처리
      - 파일 압축 풀기, 암호화
      - 복잡한 알고리즘
- 동기와 비동기
  - 동기란 무엇일까요?
    - 동기는 작업을 순서대로 하나씩 처리하는 방식
  - 비동기란 무엇일까요?
    - 비동기는 작업을 순서대로 하지 않고 여러개 동시에 처리하는 방식이다
  - 동기와 비동기 각각의 장단점은 무엇이 있을까요?
    |                      | 장점                                                 | 단점                                                                                   |
    | -------------------- | ---------------------------------------------------- | -------------------------------------------------------------------------------------- |
    | 동기(Synchronous)    | 작업 흐름이 직관적이라 설계가 간단하고 예측 가능하다 | 비효율적이다. 작업 하나가 오래 걸리면 그 작업이 끝날 때까지 화면이 멈춤                |
    | 비동기(Asynchronous) | 효율적. 시스템 자원을 효율적으로 쓸 수 있다.         | 작업 완료 순서를 예측할 수 없고 코드가 복잡해지며 오류가 발생했을 때 원인 찾기 힘들다. |
- Handler
  - Android에서 Handler란 무엇일까요?
    - 스레드 간의 통신을 담당하는 메신저이다.
    - 뒤에 나오는 Looper의 메세지들을 handler를 통해 전달함.
  - Handler를 사용하는 이유는 무엇일까요?
    - UI 작업을 메인 스레드가 아닌 워커 스레드에서 하기 위해 핸들러를 사용.
    - 모든 UI는 메인 스레드에서만 바꿀 수 있기 때문에
    - worker thread에서 main thread의 UI를 안전하게 변경하기 위해서 핸들러를 사용하는 것이다.
  - Handler의 Message란 무엇일까요?
    - 메세지란 하나의 작은 작업 단위이다.
    - 메세지는 Runnable 객체와 Message 객체 둘 중 하나로 이루어져있다.
    - 만약 Runnable 객체라면 핸들러로 메세지를 전달하지 않고 run()을 수행한다.
    - Runnable 객체가 아니라면 핸들러를 통해 처리한다.
- Looper
  - Android에서 Looper란 무엇일까요?
    - 하나의 쓰레드에 하나의 Looper가 존재한다.
    - Looper 내부에는 MessageQueue가 존재 하는데 해당 쓰레드가 처리해야할 동작이 이 큐에 저장된다.
    - 이 메세지들을 하나씩 꺼내어 Handler로 전달한다.
  - Looper를 활용하는 예시는 무엇이 있을까요?
    - Looper가 없다면 앱을 시작한 뒤 화면을 띄우고 나서 그대로 종료된다.
    - 하지만 Looper가 계속해서 돌면서 MessageQueue를 감시하다가 터치 이벤트나 화면 갱신 등의 메세지가 들어오면 즉시 Handler에 전달한다.
    - Looper가 계속해서 돌기 때문에 안드로이드 앱이 멈추지 않고 사용자와 상호작용한다.
- Coroutine
  - Coroutine이란 무엇일까요?
    - 코루틴이란 비동기 코드를 동기 코드 처럼 간결하게 작성하도록 도와주는 도구이다.
    - 경량 쓰레드라고도 불린다.
    - 쓰레드를 새로 만들지 않고 기존 쓰레드에서 작업을 잠시 멈췄다가 다시 시작하기 떄문에 효율적이다.
  - Coroutine은 언제 사용할까요?
    - Handler나 Worker Thread가 쓰이는 거의 대부분의 상황에서 사용된다.
    - 메인 쓰레드를 쓰지 않는 모든 비동기 작업에 사용된다.
      - 네트워크 통신, DB 작업, 대용량 파일 관리
  - Coroutine의 Dispatcher란 무엇일까요?
    - 해당 코루틴이 어떤 쓰레드에서 실행할 지 지정하는 역할을 한다.
  - Dispatcher의 종류에는 무엇이 있을까요?
    - Dispatchers.Default
      - CPU를 많이 사용하는 작업에 최적화된 스레드 풀에서 사용된다.
      - 무거운 연산 작업, 리스트 정렬, 복잡한 Json 파싱 작업 등
    - Dispatchers.Main
      - 안드로이드의 메인 스레드에서 코루틴을 실행한다.
      - 오직 UI와 관련된 작업에만 사용한다.
    - Dispatchers.IO
      - Input/Output 작업에 최적화 되어 있다
      - 네트워크 통신, 데이터베이스 접근, 파일 읽기/쓰기 등
