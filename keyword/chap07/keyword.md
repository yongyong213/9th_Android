- Database (DB)
    - Database의 정의는 무엇일까요?
        - 안드로이드 앱에서 데이터를 저장하고 관리하기 위한 체계적인 데이터 집합이다.
        - 안드로이드에서는 기본적으로 SQLite를 제공한다.
    - Database는 어디서 주로 사용될까요?
        - 웹이나 앱, 은행, 학교 등 모든 곳에 사용될 수 있다.
- Database Management System (DBMS)
    - DBMS란 무엇일까요?
        - Database Management System(데이터베이스 관리 시스템)
        - DB를 효율적으로 관리하고 운영하기 위한 소프트웨어이다.
    - 대표적인 DBMS는 무엇이 있을까요?
        - 관계형 DBMS : MySQL, Oracle, PostergreSQL, SQLite 등
        - 비관계형 DBMS : MongoDB, Redis 등
    - 관계형 저장 방식은 무엇일까요?
        - 데이터를 엑셀처럼 정해진 행과 열의 테이블 형식으로 저장하는 방식이다.
        - 각 행은 고유 식별자인 기본키를 가지고, 여러 테이블의 행들을 외래키를 사용하여 상호 연결할 수 있다.
        - 데이터의 구조가 정해져 있고, 데이터 간의 관계를 맺기 좋아 정형화된 데이터를 다루기 유리하다.
    - 관계형이 아닌 저장 방식에는 무엇이 있을까요?
        - NoSQL 데이터 베이스, 비관계형 데이터 베이스
        - 특정 데이터 모델에 대해 특정 목적에 맞추어 구축되어 유연하다.
            - 문서형 : JSON과 같은 문서 형식으로 데이터 저장하는 방식
            - key-value : 키, 밸류 형태로 저장하여 데이터 조회에 특화됨 캐싱, 세션 관리에 사용
            - Wide-Column : 행과 열의 테이블 구조를 사용하지만 행마다 열의 개수가 다를 수 있음, 대규모 데이터 처리에 효과적
            - 그래프 : 노드와 엣지로 구성되어 복잡한 관계를 표현하고 쿼리하는 데 강점, 추천 시스템 등에 활용
    - SQL이란 무엇일까요?
        
        SQL은 관계형 데이터베이스와 소통하기 위해 사용하는 표준 언어
        
        1. DDL(Data Definition Language)
            - 데이터 정의어
            - 데이터의 구조를 만듦
            - 데이터베이스나 테이블을 생성, 수정, 삭제
        2. DML(Data Manipulation Language)
            - 데이터 조작어
            - 테이블 안의 실제 데이터를 다루는 언어
            - 데이터 조회, 삽입, 갱신, 삭제
        3. DCL(Data Control Language)
            - 데이터 제어어
            - 데이터의 접근 권한과 보안 다룸
            - 특정 사용자에게 권한 부여, 회수
- RoomDB
    - RoomDB는 무엇일까요?
        - 스마트폰 내장 DB에 데이터를 저장하기 위해 사용하는 ORM 라이브러리이다.
        - Room은 DB를 코틀린이나 자바의 객체로 매핑해주는 역할이다.
        - 복잡한 SQL 명령어 대신 자바나 코틀린 객체를 통해 DB에 접근할 수 있다.
    - RoomDB 개발 이전에 사용하였고, 현재 RoomDB의 기반인 DBMS는 무엇일까요?
        - SQLite 이다.
        - RoomDB는 안드로이드에 기본 내장된 SQLite의 추상화 계층을 추가한 것
        - SQLite와 비교
            - Room은 컴파일 시간에 SQL 유효성 검사를 수행
            - 스키마가 바뀌었을 때 영향 받는 SQL 쿼리를 자동 업데이트
            - 추가적인 상용구 코드 없이 DB 객체를 자바 객체에 매핑할 수 있다.
    - RoomDB는 어떤 방식으로 값을 저장할까요?
        - ORM 방식 ( 객체 관계 매핑)
        1. Entity 정의
        - @Entity
            - 저장하고 싶은 클래스를 @Entity 라는 주석으로 지정하면, RoomDB가 알아서 그 객체의 표를 만듦
        - @PrimaryKey
            - 기본 키를 지정한다. 표의 열에 해당한다.
        - User라는 객체가 있으면 user라는 SQLite 테이블이 생기고 user.name은 name 열에 저장되는 것
        1. DAO 정의
        - @Dao (Data Access Object)
            - 인터페이스를 DAO로 지정하고 @Insert와 같은 데이터 수정 삽입 삭제 등의 메서드를 정의
        1. Database 정의
        - @Database 를 사용하여 클래스를 DB로 지정
